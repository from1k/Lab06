# Labaratory work 6.

Данная лабораторная работа посвещена изучению средств пакетирования на примере CPack.

## Цели и задачи.

После того, как вы настроили взаимодействие с системой непрерывной интеграции,
обеспечив автоматическую сборку и тестирование ваших изменений, стоит задуматься
о создание пакетов для измениний, которые помечаются тэгами.

Пакет должен содержать приложение solver из предыдущего задания Таким образом, каждый новый релиз будет состоять из следующих компонентов:

- архивы с файлами исходного кода (.tar.gz, .zip)
- пакеты с бинарным файлом solver (.deb, .rpm, .msi, .dmg)

Для этого нужно добавить ветвление в конфигурационные файлы для CI со следующей логикой:
если commit помечен тэгом, то необходимо собрать пакеты (DEB, RPM, WIX, DragNDrop, ...)
и разместить их на сервисе GitHub.

## Подведение итогов.
Поставленные задачи выполнены, с результатами можно ознакомиться либо в самом репозитории, либо во вкладке [releases](https://github.com/from1k/Lab06/releases).

## Вывод

В ходе выполнения работы был полностью настроен проект solver: создана правильная структура с библиотеками и приложениями, написаны CMake файлы 
для сборки и генерации установочных пакетов через CPack, обновлён `.gitignore`. Репозиторий инициализирован и запушен на GitHub с тэгом `v1.0.0`. Освоены навыки настройки CMake/CPack.
